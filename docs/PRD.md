## 1. 产品概述

### 1.1 产品定位
ScreenSight 是一款运行于 Windows 平台的**屏幕时间线工具**，核心能力是通过定时截取屏幕并交由视觉大模型（VLM）分析，**细粒度地识别用户在某一时刻正在做什么**，并基于识别结果生成小时报/日报/周报/月报，让用户清晰知晓自己的时间花在了哪里。

### 1.2 核心价值（解决痛点）
- **行为识别要细**：不只是"工作/娱乐"的粗分类，而是细化到"在 VSCode 写 Python 代码，项目是 ScreenSight，正在修改截图模块"这种程度。
- **时间去向可视化**：通过小时报/日报/周报/月报，让用户清晰看到哪些时段把时间花在了哪里，这是该产品的关键痛点解决方案。
- **可检索可回溯**：支持传统关键词搜索与 RAG 问答检索，配合多维筛选，能快速定位历史活动。

### 1.3 目标用户
**个人自用**。主要面向希望管理个人时间、提升效率的知识工作者/开发者。产品以单机本地工具形态交付，不涉及团队/企业多用户场景。

### 1.4 平台
- 操作系统：Windows 10 / 11（x64）
- 部署形态：**后台服务 + 本地 Web 界面**
  - 后台服务常驻系统托盘，负责截屏、VLM 调用、存储、报告生成等
  - 用户点击托盘图标打开本地浏览器（访问 `localhost`）查看时间线/报告/搜索
- 启动方式：**手动启动**（不自启），用户主动运行程序后开始记录

## 2. 核心工作流程

```
[键鼠活动检测] → [活跃时每30秒截屏]
        │              │
        │              ├─ 焦点屏高清图 → 云端VLM识别 → 结构化结果(类别+描述+对象+置信度)
        │              ├─ 其余屏 → 仅存档不识别
        │              └─ 存档图 → 低质量压缩存储
        │
        │   [识别结果即时向量化] → [向量库]
        │
        ├─ [同类活动合并为时段]
        │
        └─ [定时/手动生成报告] → 小时报/日报/周报/月报
                                   ↑
                            规则统计(准确) + LLM润色(自然语言)

[搜索/问答] → 关键词匹配 / RAG问答 → 时间+类目+项目+置信度多维筛选
```

## 3. 功能需求

### 3.1 截屏模块

#### 3.1.1 截屏策略
- **活跃期间隔**：用户有键鼠活动时，默认每 **30 秒**截屏一次。
- **多屏处理**：多显示器场景下，**只识别当前焦点屏**（活动窗口所在显示器）；其余屏幕仅截屏存档，不送 VLM，以节省 API 费用。
- **图片质量分离**：
  - 送 VLM 的图：高质量原图（保证识别准确度）
  - 本地存档图：低质量压缩存储（节省磁盘）
  - 两者分离存储，存档图不可用于事后重新验证 VLM 识别结果（已知取舍）

#### 3.1.2 静默与锁屏处理
- **在线判断依据**：**以键鼠活动判断用户是否在线**，不依赖画面变化。
  - 键鼠活动 → 在线 → 维持 30 秒间隔
  - 键鼠无活动超过阈值 → 进入空闲态 → 拉长截屏间隔（具体阈值与拉长后的间隔在 ARCHITECTURE 中细化，初定 5 分钟无活动 → 间隔拉长到 5 分钟）
  - 键鼠恢复 → 立即回到 30 秒间隔
- **锁屏暂停**：检测到 Windows 会话锁屏/睡眠状态时，**完全停止截屏**，避免产生黑屏/锁屏图浪费存储与 API 费用；解锁后恢复。
- **空闲标记**：静默时段在时间线上标记为"空闲/离开"状态，而非持续记录同一活动。

#### 3.1.3 手动控制
- 系统托盘提供**手动暂停/恢复**按钮，用户可随时暂停记录（如开会、休息时）。

### 3.2 VLM 行为识别模块

#### 3.2.1 云端 VLM 来源
- 采用**云端 API**，遵循 **OpenAI 兼容协议**（base_url + api_key），可接 OpenAI / 通义千问 VL / Claude 代理 / DeepSeek 等绝大多数厂商。
- API Key 由**用户自带**，在设置中配置；工具本身不内置服务、不代收费用。

#### 3.2.2 识别输出结构
每次识别输出以下结构化数据：
| 字段 | 说明 |
|---|---|
| 一级类别 | 预设的 23 类之一（见 3.2.3） |
| 二级描述 | VLM 动态生成的细粒度描述，如"Python-ScreenSight项目-修改截图模块" |
| 对象/项目名 | 如 ScreenSight、某影片名、某文档主题 |
| 活动描述 | 一句话自然语言描述当前行为 |
| 置信度 | 0~1 的浮点，表示识别把握程度 |

#### 3.2.3 一级分类体系（预设 23 类，分 6 组）

**工作生产**
1. 编码开发 — IDE/编辑器中编写代码
2. 代码审查 — Review PR / Diff / 阅读他人代码
3. 调试排错 — 调试、看日志、排查问题
4. 终端操作 — 命令行、Shell、脚本执行
5. 文档撰写 — Word/Markdown/Wiki/Notion 写作
6. 文档阅读 — PDF/Word/电子书/报告阅读
7. 技术资料查阅 — API 文档/技术博客/StackOverflow
8. 笔记知识整理 — 笔记、知识库整理、思维导图

**设计创作**
9. UI/UX 设计 — Figma/Sketch/Axure
10. 图像编辑 — Photoshop/插画/修图
11. 音视频制作 — 剪辑/特效/音频处理

**沟通协作**
12. 即时通讯 — 微信/Slack/Discord/QQ
13. 邮件处理 — 邮件客户端
14. 视频会议 — Zoom/Teams/腾讯会议

**信息浏览**
15. 网页浏览 — 浏览器资讯/搜索
16. 社交媒体 — 微博/Twitter/Reddit/小红书

**娱乐休闲**
17. 视频/电影 — B站/Netflix/本地播放器
18. 音乐/播客 — 网易云/Spotify
19. 游戏 — 各类游戏

**学习提升**
20. 在线学习 — 网课/教程/慕课

**系统其他**
21. 文件管理 — 资源管理器
22. 系统工具 — 设置/工具软件
23. 其他/空闲

> 二级分类由 VLM 根据画面细节动态生成，不预设固定枚举。

#### 3.2.4 连续活动合并
- 连续多张截图被识别为同一类活动时，**合并为一条时间段记录**（如 9:00-9:25 写代码），避免时间线重复。
- 合并判定规则：一级类别相同 + 对象/项目名相同（或连续无中断）→ 合并；具体合并算法在 ARCHITECTURE 细化。

#### 3.2.5 低置信度处理
- VLM 返回置信度低于阈值（初定 0.6）时：**仍自动按结果归类**，但在记录上打"**低置信**"标记。
- 报告与搜索中可按置信度筛选，过滤掉低置信记录。

### 3.3 存储模块

#### 3.3.1 数据存储方案
- **结构化数据**（识别结果、时段、报告、配置等）：**SQLite**
- **向量数据**（用于 RAG 检索）：**sqlite-vec**（SQLite 向量插件，单文件好备份迁移）
- **截图文件**：文件系统存储，按日期分目录组织

#### 3.3.2 截图保留策略（三级梯度）
为平衡磁盘占用与可回溯性，采用三级梯度保留：

| 阶梯 | 时间范围 | 截图保留策略 |
|---|---|---|
| 近期 | 0 ~ 30 天 | 保留压缩后的存档原图 |
| 中期 | 30 天 ~ 3 个月 | 降质量再次压缩（更小体积） |
| 远期 | 3 个月以上 | 仅保留 VLM 识别结果文本，删除截图文件 |

> 具体压缩参数（质量、尺寸）在 ARCHITECTURE 中细化。报告与搜索功能不依赖远期截图，仅依赖识别结果文本。

#### 3.3.3 数据备份与迁移
- 提供**一键导出**整个数据目录（数据库 + 截图）为压缩包。
- 提供**从备份恢复**功能。

#### 3.3.4 隐私擦除
- 用户可在时间线选中某时段或某条记录，**手动删除**其截图与识别结果（向量库同步删除）。

### 3.4 报告模块

#### 3.4.1 报告生成方式
- **规则统计 + LLM 润色**：
  - 规则部分严格计算时长、占比、分类统计、Top 排行等**数字**，保证准确，避免 LLM 幻觉。
  - LLM 部分基于规则统计结果，生成自然语言总结与趋势洞察。
- 报告类型：**小时报 / 日报 / 周报 / 月报**。

#### 3.4.2 自动生成时机
- 每小时末自动生成当小时报告
- 每日结束自动生成日报（如 23:00 或次日首次开机）
- 每周一自动生成上周周报
- 每月初自动生成上月月报
- 同时支持用户**手动触发**生成任意时段报告

#### 3.4.3 报告内容
每份报告尽可能丰富，包含：
1. **分类时长与占比**：各一级类别的时长与百分比（如开发 4h/50%、娱乐 2h/25%）
2. **时间轴活动列表**：按时间轴列出主要活动时段（如 9:00-11:30 写代码 ScreenSight）
3. **LLM 总结与洞察**：自然语言总结与趋势分析（如"本周开发占比环比上升 20%"）
4. **Top 项目/活动排行**：同一活动/项目累计时长 Top N

#### 3.4.4 报告导出
- 界面直接查看（支持翻阅历史报告）
- 导出为 **Markdown**
- 导出为 **PDF**

### 3.5 搜索与 RAG 模块

#### 3.5.1 传统关键词搜索
- **搜索范围**：VLM 输出的全部内容，包括：
  - 识别描述文本
  - 报告文本
  - 分类标签与对象名
- 不做截图 OCR（已知取舍，避免额外 OCR 成本）。

#### 3.5.2 RAG 问答检索
- **用途**：问答式检索。用户用自然语言提问（如"上周我在哪个项目上花时间最多""最近什么时候看过某影片"），系统检索相关历史记录后由 LLM 回答。
- **向量化时机**：VLM 识别后**即时向量化**识别描述文本，存入向量库。

#### 3.5.3 搜索结果筛选
搜索结果支持以下多维筛选：
1. **时间范围筛选**：全时段 / 近一周 / 近一个月 / 自定义起止日期
2. **类目筛选**：按一级/二级分类筛选
3. **项目/对象筛选**：按对象/项目名筛选
4. **置信度筛选**：按置信度筛选（如只看高置信记录）

### 3.6 时间线界面

#### 3.6.1 呈现方式
- **时间轴 + 详情列表**组合：
  - 上方：以一天为横轴的色块时间轴，不同颜色表示不同活动时段，鼠标悬停看概要，点击看详情
  - 下方：选中时段的详细记录列表，含缩略图、时间、描述、置信度等

#### 3.6.2 交互
- 点击时间轴色块 → 下方列表展示该时段详情
- 支持按天/周/月切换时间范围
- 支持手动删除某时段/某条记录（隐私擦除入口）

## 4. 非功能需求

### 4.1 隐私与安全
- **纯本地存储不上云**：截图、识别结果、向量数据全部存本地，绝不外传。
- **云端 VLM 临时上传不留存**：截图临时上传云端做识别，识别后云端不留存（依赖云厂商不留存承诺），本地是唯一持久化存储。
- 不修改 `.env` 中的真实敏感值；配置示例使用占位符（如 `<API_KEY>`）。

### 4.2 性能
- 后台服务内存占用控制在合理范围（目标 < 300MB，不含浏览器）。
- 截屏与 VLM 调用异步进行，不阻塞用户操作。
- VLM 识别失败时本地重试有限次，仍失败则记录错误并跳过，不中断主流程。

### 4.3 可靠性
- VLM 调用失败、网络异常时降级处理：保留截图待识别，标记"识别失败"，不影响后续截屏。
- 数据库操作事务化，避免部分写入导致数据不一致。

### 4.4 技术栈
- **后台服务**：Python（mss 截图 / Pillow 图像处理 / httpx 调云 API / sqlite-vec 向量检索）
- **本地 Web 后端**：FastAPI
- **前端**：React 或 Vue（轻量 SPA）
- **打包**：Windows 可执行（后续确定打包方案）

## 7. 验收标准（MVP）

由于选择"全功能一次交付"，MVP 须包含全部功能模块且可端到端运行：

1. **截屏**：能定时（30秒）截取焦点屏，锁屏暂停、空闲拉长间隔、手动暂停/恢复均生效
2. **识别**：调用用户配置的云 VLM，输出结构化结果（类别+描述+对象+置信度），连续同类合并为时段
3. **存储**：SQLite + sqlite-vec 落库，截图三级梯度保留，支持手动删除与备份导出/恢复
4. **报告**：小时报/日报/周报/月报按时机自动生成 + 手动触发，规则统计准确，LLM 润色，可导出 MD/PDF
5. **搜索**：关键词搜索（识别描述+报告+标签对象名）+ RAG 问答，支持时间/类目/项目/置信度筛选
6. **时间线**：色块时间轴 + 详情列表，可翻阅历史、删除记录
7. **费用统计**：展示调用次数与预估费用

> 各模块的人工验证现象与通过结论将记录于 `docs/process.md`。
