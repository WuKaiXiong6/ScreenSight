## 1. 产品概述

### 1.1 产品定位
ScreenSight 是一款运行于 Windows 的**屏幕时间线工具**。它通过定时截取焦点屏并交由云端视觉大模型（VLM）识别，**细粒度还原"某一时刻用户在做什么"**，并基于识别结果生成小时报/日报/周报/月报，让用户清晰知晓时间花在了哪里。

### 1.2 核心价值
- **行为识别要细**：不只是"工作/娱乐"粗分类，而是细化到"在 VSCode 写 Python，项目 ScreenSight，正在改截图模块"。
- **时间去向可视化**：通过小时报/日报/周报/月报，让用户看到哪些时段把时间花在了哪里——这是本产品解决的核心痛点。
- **可检索可回溯**：关键词搜索 + RAG 问答，配合多维筛选，快速定位历史活动。

### 1.3 目标用户
**个人自用**。面向希望管理个人时间、提升效率的个人。以单机本地工具形态交付，不涉及团队/企业多用户。

### 1.4 平台与部署
- 操作系统：Windows 10 / 11（x64）
- 形态：**后台服务 + 本地 Web 界面**
  - 后台服务常驻系统托盘，负责截屏、VLM 调用、存储、报告生成
  - 点击托盘图标打开本地浏览器访问时间线/报告/搜索
- 启动：**手动启动**（不自启），用户主动运行后开始记录

### 1.5 非目标（Out of Scope）
明确不做，避免需求蔓延：
- ❌ 团队/企业多用户、协作、共享
- ❌ 移动端、macOS、Linux
- ❌ 截图 OCR（避免额外成本，已知取舍）
- ❌ 截图本地 VLM 识别（隐私更强但需强 GPU，质量受限）
- ❌ 限额提醒/自动节流（费用仅统计展示，用户自行关注云账单）
- ❌ 记录自动开始（程序启动后默认不立即记录，由用户或设置决定开始时机；托盘本身支持配置开机自启，但自启后不等于自动记录）

## 2. 使用场景

| 场景 | 用户行为 | 产品响应 |
|---|---|---|
| 复盘今天 | 晚上打开时间线 | 看到一天色块轴，点时段看详情，了解时间分布 |
| 周度复盘 | 周一看上周报告 | 自动生成的周报：分类占比、Top 项目、LLM 趋势洞察 |
| 找某次活动 | "上周三下午看的那部电影叫什么" | RAG 问答检索，返回带来源的答案 |
| 演示分享 | 给别人看报告 | 开隐私模式：对象名打码、截图模糊、导出脱敏 |
| 隐私擦除 | 某时段不想留痕 | 时间线选中时段删除，截图+识别+向量同步清除 |

## 3. 核心工作流程

```
[键鼠活动检测] → [活跃时每 30 秒截屏]
        │              │
        │              ├─ 焦点屏高清图 → 云端 VLM → 结构化结果
        │              ├─ 其余屏 → 仅存档不识别
        │              └─ 存档图 → 低质量压缩存储
        │
        │   [识别结果即时向量化] → [向量库]
        │
        ├─ [同类活动合并为时段]
        │
        └─ [定时/手动生成报告] → 小时报/日报/周报/月报
                                   ↑
                            规则统计(准确) + LLM 润色(自然语言)

[搜索/问答] → 关键词匹配 / RAG 问答 → 时间+类目+项目+置信度多维筛选
```

> 截屏间隔、状态机、合并算法等实现细节见 ARCHITECTURE.md。

## 4. 功能需求

### 4.1 截屏与记录控制

#### 4.1.1 截屏策略
- **活跃期间隔**：用户有键鼠活动时，默认每 **30 秒**截屏一次（可在设置中调整）。
- **多屏处理**：多显示器场景下，**只识别当前焦点屏**（活动窗口所在显示器）；其余屏仅截屏存档，不送 VLM，以节省费用。
- **图片质量分离**：
  - 送 VLM 的图：高质量原图（保证识别准确度）
  - 本地存档图：低质量压缩存储（节省磁盘）
  - 存档图不可用于事后重新验证 VLM 识别结果（已知取舍）

#### 4.1.2 在线判断与静默
- **在线判断依据**：以键鼠活动判断用户是否在线，不依赖画面变化。
  - 键鼠活动 → 在线 → 维持 30 秒间隔
  - 键鼠无活动超阈值 → 进入空闲态 → 拉长截屏间隔（默认 5 分钟无活动 → 间隔拉长到 5 分钟）
  - 键鼠恢复 → 立即回到 30 秒间隔
- **锁屏暂停**：检测到 Windows 会话锁屏/睡眠时，**完全停止截屏**，避免黑屏/锁屏图浪费存储与费用；解锁后恢复。
- **空闲标记**：空闲/锁屏时段在时间线上标记为"空闲/离开"**状态**（非活动类别），不计入活动分类占比，单独展示。

#### 4.1.3 手动控制
- 系统托盘提供**手动暂停/恢复**，用户可随时暂停记录（如开会、休息）。

### 4.2 VLM 行为识别

#### 4.2.1 云端 VLM 来源
- 采用**云端 API**，兼容 OpenAI / 通义千问 VL / Claude 代理 / DeepSeek 等主流厂商。
- API Key 由**用户自带**，在设置中配置；工具本身不内置服务、不代收费用。

#### 4.2.2 识别输出结构
每次识别输出结构化数据：

| 字段 | 说明 |
|---|---|
| 一级类别 | 从当前一级类别列表中选择（默认"工作""娱乐"，用户可在设置中扩展） |
| 二级描述 | VLM 动态生成的细粒度描述（见 4.2.3） |
| 对象/项目名 | 如 ScreenSight、某影片名、某文档主题 |
| 活动描述 | 一句话自然语言描述当前行为 |
| 置信度 | 0~1 浮点，表示识别把握程度 |

#### 4.2.3 分类体系（一级极简 + 二级 AI 动态）

##### 一级类别：极简 + 可扩展
- **默认两类**：`工作` / `娱乐`。
- **用户可扩展**：用户可在设置中预设新的一级类别（如 `学习`、`生活`）。VLM 只能从"默认 + 用户扩展"的列表中选择，**不得自由造词**，避免出现"工作/上班/办公"这类同义重复。
- 设计意图：一级只做最粗划分，细粒度交给二级。

##### 二级描述：VLM 动态生成（按类型分别给规则）
二级由 VLM 根据画面动态生成，**不预设固定枚举**，但生成时遵循按一级类别区分的规则：

| 一级类别 | 二级生成规则 | 示例 |
|---|---|---|
| 工作（开发场景） | **优先以项目名分类** + 具体在做什么 | `ScreenSight-改截图模块`、`Talk2ESP-调试串口通信` |
| 工作（其他） | 内容主题 + 动作 | `API文档-查阅OpenAI兼容协议`、`周报-撰写本周进展` |
| 娱乐 | 内容类型 + 名称/主题 | `游戏-黑神话悟空`、`视频-吃播`、`音乐-周杰伦` |
| 用户自定义类别 | VLM 按该类别的语义生成对应细粒度描述 | （取决于用户定义） |

> 提示词需明确要求 VLM：开发场景从标题栏/文件栏/代码内容推断项目名与当前动作；娱乐场景识别内容类型与具体名称。

##### 模糊场景：以内容为主、平台为辅，动态识别
同一平台的不同内容可能分到不同一级类别，**不写死平台→一级映射**，由提示词引导 VLM 依据内容本身判定：

| 场景 | 一级 | 二级（示例） |
|---|---|---|
| B站看技术教程 | 工作 | `B站-技术教程-Python异步` |
| B站看吃播/vlog | 娱乐 | `B站-吃播` |
| 浏览器查 API 文档 | 工作 | `API文档-OpenAI兼容协议` |
| 浏览器刷微博 | 娱乐 | `社交媒体-微博` |
| 看技术书 PDF | 工作（或用户自定义"学习"） | `技术书-Python并发编程` |
| 看小说 | 娱乐 | `小说-三体` |

> 提示词不应硬编码"B站=娱乐"这类规则，而应描述判定倾向："以画面呈现的内容本身为主，平台仅为线索之一"。

#### 4.2.4 连续活动合并
- 连续多张截图被识别为同一类活动时，**合并为一条时间段记录**（如 9:00-9:25 写代码），避免时间线重复。
- 合并判定：**一级类别相同 + 二级描述相同**（或连续无中断）→ 合并。算法见 ARCHITECTURE.md。

#### 4.2.5 低置信度处理
- VLM 返回置信度低于阈值（默认 0.6）时：**仍自动按结果归类**，但打"**低置信**"标记。
- 报告与搜索中可按置信度筛选，过滤掉低置信记录。

### 4.3 数据与隐私

#### 4.3.1 存储原则
- **纯本地存储不上云**：截图、识别结果、向量数据全部存本地，绝不外传。
- **云端 VLM 临时上传不留存**：截图临时上传云端识别，识别后云端不留存（依赖云厂商承诺），本地是唯一持久化存储。

#### 4.3.2 截图保留策略（三级梯度）
为平衡磁盘占用与可回溯性：

| 阶梯 | 时间范围 | 截图保留策略 |
|---|---|---|
| 近期 | 0 ~ 30 天 | 保留压缩后的存档原图 |
| 中期 | 30 天 ~ 3 个月 | 降质量再次压缩 |
| 远期 | 3 个月以上 | 仅保留识别结果文本，删除截图文件 |

> 报告与搜索功能不依赖远期截图，仅依赖识别结果文本。压缩参数见 ARCHITECTURE.md。

#### 4.3.3 隐私擦除
- 用户可在时间线选中某时段或某条记录，**手动删除**其截图与识别结果（向量库同步删除）。

#### 4.3.4 隐私展示模式
- 顶部开关开启后：
  - 对象名打码（首字符 + 星号）
  - 截图 CSS 模糊
  - 关闭图片预览
  - 导出 Markdown 时对象名脱敏
- 自用时可关闭，显示完整信息。

#### 4.3.5 备份与迁移
- 一键导出整个数据目录（数据库 + 截图）为压缩包。
- 从备份恢复。

### 4.4 报告

#### 4.4.1 生成方式
- **规则统计 + LLM 润色**：
  - 规则部分严格计算时长、占比、分类统计、Top 排行等数字，保证准确，避免 LLM 幻觉。
  - LLM 部分基于规则统计结果，生成自然语言总结与趋势洞察。
- 类型：**小时报 / 日报 / 周报 / 月报**。

#### 4.4.2 自动生成时机
- 每小时末自动生成当小时报告
- 每日结束自动生成日报（次日首次启动时补生成）
- 每周一自动生成上周周报
- 每月初自动生成上月月报
- 同时支持用户**手动触发**生成任意时段报告

#### 4.4.3 报告内容
1. **分类时长与占比**：按一级类别（工作/娱乐/…）与二级描述两个粒度统计时长与百分比（如 工作 6h/75%，其中 ScreenSight-改截图模块 3h）
2. **时间轴活动列表**：按时间轴列出主要活动时段
3. **LLM 总结与洞察**：自然语言总结与趋势分析
4. **Top 项目/活动排行**：同一二级描述累计时长 Top N

#### 4.4.4 报告导出
- 界面直接查看（支持翻阅历史报告）
- 导出 Markdown（支持脱敏参数）
- 导出 PDF

### 4.5 搜索与 RAG

#### 4.5.1 关键词搜索
- **范围**：VLM 输出全部内容（识别描述文本、报告文本、分类标签与对象名）。
- **不做截图 OCR**（已知取舍）。

#### 4.5.2 RAG 问答检索
- **用途**：问答式检索。用户自然语言提问（如"上周哪个项目花时间最多""最近什么时候看过某影片"），系统检索相关历史记录后由 LLM 回答。
- **向量化时机**：VLM 识别后即时向量化识别描述文本，存入向量库。
- **retrieve_only 模式**：跳过 LLM 生成，仅返回检索到的原始记录（用于节省 token 或 LLM 不可用时）。

#### 4.5.3 搜索结果筛选
1. 时间范围：全时段 / 近一周 / 近一月 / 自定义起止
2. 类目：按一级/二级分类
3. 项目/对象：按对象/项目名
4. 置信度：按置信度筛选（如只看高置信记录）

### 4.6 时间线界面

#### 4.6.1 呈现
- **时间轴 + 详情列表**：
  - 上方：以一天为横轴的色块时间轴，不同颜色表示不同活动时段，鼠标悬停看概要，点击看详情
  - 下方：选中时段的详细记录列表，含缩略图、时间、描述、置信度

#### 4.6.2 交互
- 点击时间轴色块 → 下方列表展示该时段详情
- 按天/周/月切换时间范围
- 手动删除某时段/某条记录（隐私擦除入口）
- **首屏回退**：若今天无数据，自动回退到最近有数据的日期，避免空白被误判为故障
- **hash 路由**：支持 `#/timeline?date=YYYY-MM-DD` 深链到某天，刷新不丢失当前页
- **时段详情前后翻**：选中某时段后，可在详情视图内左右切换浏览前后相邻时段，无需反复返回点列表

#### 4.6.3 当前时刻指针
- 当查看"今天"时，时间轴上显示当前时刻的指针（竖线），页面打开时自动滚动到指针附近，无需手动查找当前位置。
- 查看历史日期时不显示指针。

#### 4.6.4 多天对比视图
- 选择 2~7 天，多行色块轴上下排列，直观对比同一时段每天在做什么（如"本周每天下午 3 点都在干嘛"）。
- 支持按一级类别着色统一，便于跨天对比分布。

### 4.7 设置

设置以"给用户尽可能多的自主选择权"为目标，覆盖以下维度：

#### 4.7.1 界面外观
- **主题模式**：亮色 / 暗色 / 跟随系统。
- **字号大小**：小 / 中 / 大，适配不同视力与屏距。
- **色块配色方案**：时间轴色块的配色规则可配（如按一级类别着色、或按二级着色、或自定义颜色映射）。
- **界面语言**：中文 / 英文切换。

#### 4.7.2 通知与启动
- **托盘通知开关**：识别失败 / 磁盘不足等异常是否弹托盘提示。
- **开机自启托盘**：托盘程序可配置开机自启（注意：托盘自启 ≠ 记录自启，记录是否开始由 4.7.4 的"自启动记录时机"控制）。
- **窗口关闭行为**：关闭窗口时 最小化到托盘 / 退出程序，二选一。
- **检查更新开关**：开启后界面内一键检查新版本，关闭则不查。

#### 4.7.3 时区与数据展示
- **时区**：默认北京时间，可选其他时区，影响时间轴与报表的时间展示。
- **时间格式**：12h / 24h，是否含秒。
- **周起始日**：周一 / 周日，影响周报表边界。
- **默认页与默认报告**：启动后默认进入的页面、默认查看的报告类型。

#### 4.7.4 截屏控制
- **截屏间隔/空闲阈值可调**：活跃截屏间隔、空闲阈值、空闲态间隔均可调，适配不同费用与精度需求。
- **多屏截屏选择**：配置哪些显示器参与截屏（如只截主屏、或排除某屏）。
- **排除指定应用**：配置不截屏的应用窗口（如密码管理器、银行页面），命中即跳过截屏与识别。
- **自启动记录时机**：程序启动后是否立即开始记录、还是等待用户手动开始。

#### 4.7.5 识别与分类
- **一级类别增删**：用户可在设置中增删一级类别（默认"工作""娱乐"，可加"学习""生活"等），增删后 VLM 识别的可选范围同步更新。
- **自定义提示词**：用户可编辑识别提示词模板（如追加"项目名优先从标题栏推断"等指令），供高级用户调优识别效果。
- **低置信度阈值与处理策略**：低置信度阈值（默认 0.6）可调；可选低置信记录是仍归类统计、还是仅标记不统计。
- **厂商预设**：内置常见 VLM/LLM 厂商预设（火山方舟 / 小米 / OpenAI 等），选预设后仅需填 API Key，降低配置门槛。

#### 4.7.6 报告
- **报告生成时机可调**：日报/周报/月报的自动生成时间点可调，并可独立开关各类型自动生成。

#### 4.7.7 数据与隐私
- **保留期可调**：截图三级梯度的阈值天数（默认 30 / 90 天）可调。
- **按范围批量清除**：一键清空某时间范围的所有记录（如清空近 30 天娱乐记录），含截图、识别结果、向量同步删除。
- **存储清理工具**：手动或定时触发数据库 VACUUM、清理孤儿截图文件。
- **备份导出选项**：导出时可选择是否含截图、导出格式，满足不同备份需求。

#### 4.7.8 RAG 与检索
- **检索条数 K**：RAG 检索返回的 top-K 可调（默认如 5，可调 1~20）。
- **embedding 模型切换**：本地 embedding 模型可切换（如 bge-large-zh / bge-m3），供高级用户选择精度与体积的平衡。
- **向量库重建**：更换 embedding 模型或调整提示词后，可一键重建已有数据的向量。
